# What Was The Goal Of Day 4?

The goal was to build a reusable retry framework that can:

* Automatically recover from transient failures
* Use exponential backoff
* Log retry attempts
* Support typed configuration
* Hide Tenacity internals
* Be reusable across all future ETL projects


```python id="vj8x0m"

```

# What Is Jitter?

Without jitter:

```text id="5s1jpk"
Service A
Service B
Service C

Fail at same time
```

All retry:

```text id="db2ptj"
1 second
2 seconds
4 seconds
```

at exactly the same moment.

This can overload the service.

---

# With Jitter

Instead:

```text id="5xvk8e"
1.2 sec
1.8 sec
2.3 sec
```

Retries become randomized.

Much safer.

# Advanced Typing Concepts Learned

Day 4 introduced some advanced Python typing concepts.

---

# ParamSpec

We used:

```python id="jlwmzg"
P = ParamSpec("P")
```

---

## Why?

Decorators often destroy function signatures.

Without ParamSpec:

```python id="jlwmzh"
def wrapper(*args, **kwargs)
```

loses type information.

---

## Example

Original:

```python id="jlwmzi"
def fetch_data(
    user_id: int
) -> str:
```

After decorator:

Type checker still understands:

```python id="jlwmzj"
user_id: int
```

This is very powerful.

---

# TypeVar

We used:

```python id="jlwmzk"
R = TypeVar("R")
```

---

## Why?

Preserves return type.

Example:

```python id="jlwmzl"
def fetch_data() -> str
```

remains:

```python id="jlwmzm"
str
```

after decoration.

---

# Real Engineering Impact

Without ParamSpec + TypeVar:

```text id="jlwmzn"
Autocomplete breaks
Type checking breaks
IDE support degrades
```

---

# Error 3

```text id="jlwmzt"
Item "None" of "Future | None"
has no attribute "exception"
```

---

## Why?

Tenacity defines:

```python id="jlwmzu"
retry_state.outcome
```

as:

```python id="jlwmzv"
Future | None
```

Mypy demanded proof that it wasn't None.

---

## Fix

```python id="jlwmzw"
if retry_state.outcome is None:
    return
```

---

## Lesson

Static analysis requires explicit safety checks.

---

# Error 4

```text id="jlwmzx"
Item "None" of "WrappedFn | None"
has no attribute "__name__"
```

---

## Why?

Tenacity types:

```python id="jlwmzy"
retry_state.fn
```

as:

```python id="jlwmzz"
WrappedFn | None
```

---

## Fix

```python id="jlwmz10"
retry_state.fn.__name__
if retry_state.fn is not None
else "unknown"
```

---

## Lesson

Type systems force us to handle edge cases.

---

# Error 5

```text id="jlwmz11"
Redundant cast
```

---

## Why?

We added:

```python id="jlwmz12"
cast(R, ...)
```

assuming Tenacity needed it.

Current typings already inferred the correct type.

---

## Fix

Removed the cast.

---

## Lesson

Don't overuse casts.

Use them only when truly necessary.
