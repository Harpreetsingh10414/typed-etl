# Traditional Logging Problem

Most beginners write:

```python id="l08p1w"
print("file loaded")
print("database connected")
print("api failed")
```

Output:

```text id="4fjlwm"
file loaded
database connected
api failed
```

Problems:

* No timestamp
* No severity
* No tracking
* No searchability
* No metadata

In production, this is almost useless.

---

# Structured Logging

Instead we built:

```python id="4ft16i"
logger.info(
    "file_loaded",
    filename="employees.csv",
    rows=1000,
)
```

Output:

```json id="g1u3o9"
{
  "event": "file_loaded",
  "filename": "employees.csv",
  "rows": 1000,
  "timestamp": "...",
  "level": "info"
}
```

This is called:

```text id="c43d34"
Structured Logging
```

because every log has structure.

---


With structured logs:

Easy.

# 3. bind_correlation_id()

Most important concept of Day 3.

---

# Problem

Imagine:

```text id="a6wl8n"
Pipeline A
Pipeline B
Pipeline C
```

running simultaneously.

Logs:

```text id="m44b2g"
Started
Started
Started
API Failed
CSV Loaded
Finished
```

Which pipeline generated which log?

Impossible to know.

---

# Solution

Generate:

```text id="3jlp6g"
abc123
```

and attach it everywhere.

Example:

```python id="ndxz6i"
bind_correlation_id()
```

creates:

```text id="e83d71"
38ec70c3-6a69-460e-8292-30d38747ddeb
```

---

# Result

Every log contains:

```json id="n80lj8"
{
  "correlation_id": "38ec70..."
}
```

Now we can search:

```text id="1twr8g"
correlation_id=38ec70...
```

and see the complete execution flow.

---

# 4. log_context()

Purpose:


---

# Problem

Without context:


# Solution

Context Binding.

Write once:


Everything inside automatically gets:

```json id="y58yzv"

  "job_name": "employee_ingestion"
}
```

Instead of passing:

to every log,

Huge productivity gain.

# Error 1

```text id="lf1u38"
Returning Any from function declared to return BoundLogger
```

---

## Why?

Structlog typing isn't perfect.

Mypy saw:

```python id="9h7oaq"
Any
```

instead of:

```python id="uzhhx5"
BoundLogger
```

---

## Fix

```python id="xq4eh7"
cast(
    BoundLogger,
    structlog.get_logger()
)
```

---

## Lesson

Static type checkers sometimes need help understanding third-party libraries.

---

# Error 2

```text id="0trte5"
PrintLogger has no attribute name
```

---

## Why?

Structlog created:

```text id="m6tspq"
PrintLogger
```

but:

```python id="3tl19l"
add_logger_name
```

expects:

```text id="bxfjr0"
logging.Logger
```

---

## Fix

```python id="ctg82t"
logger_factory=structlog.stdlib.LoggerFactory()
```

and

```python id="jvjlwm"
wrapper_class=structlog.stdlib.BoundLogger
```

---

## Lesson

Always understand how libraries are wired internally.

Don't blindly copy code.

# Interview Questions You Can Now Answer

### Why structured logging over print statements?

Because logs become machine-readable, searchable, and analyzable.

---

### What is a correlation ID?

A unique identifier attached to every log/event belonging to a single workflow execution.

---

### Why context-based logging?

To avoid repeatedly passing the same metadata to every log statement.

---

### Why JSON logs?

Monitoring platforms parse JSON efficiently and allow powerful querying.

---

### What is observability?

The ability to understand internal system behavior through external outputs like logs, metrics, and traces.
